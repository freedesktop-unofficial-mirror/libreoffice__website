This is however the idl file and we need a header file that our implementation
can then use, so what is necessary is first a 'build' in the offapi/ directory
that then updates its database of types/interfaces that is then used by offuh/
to generate header files that we can use. In order that offuh/ can have the
updated types database, we 'deliver' the new set from offapi/ and then 'build'
in offuh/ which generates the XHelloWorldDialog.hpp file that we can use,
'deliver' then copies them over to solver/ and from then on the header file is
available to all other projects beyond.
