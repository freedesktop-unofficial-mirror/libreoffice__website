ooo-build-1-3 and the rest from Gentoo). But AFAICT the only commit in
ooo-build-1-3 after 1.3.7 which touches the psprint module is the patch

Last night I did exactly the same build again except that I disabled the
psprint-m46-to-m62.diff patch (+ tiny fixes to make it build again), so
the psprint module was again in the same state as in 1.3.6 (there the
psprint-m46-to-m53.diff was _NOT_ applied). This disabled the threaded
cups query on startup and now my openoffice installation recognizes my
cups printers again.
