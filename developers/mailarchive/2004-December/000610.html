I've created a child workspace (gcj1) for gij/gcj and openoffice. I'm
been able to get helpcontent2 and configmgr built with gcj so I'm fairly
happy with that as its the major hurdle. Some other modules don't fully
build with gcj, generally because of use of sun java sun.net.* classes
which don't exist in gcj along with some other classes which gcj doesn't
have yet, so I'm currently simply finding and conditionally not building
the troublemakers when using gcj like --without-java does for all java
stuff.

My suggested config_office changes for configure turn 
--enable-java / --disable-java into --with-java / --without-java and 
then additionally to force building with gij/gcj use --with-java=gij

So to play with the build grab SRC680_m63 and the gcj workspace tag is
cws_src680_gcj1.  Add --with-java=gij to your configure line.
