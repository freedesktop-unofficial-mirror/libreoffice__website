> 
> 	So - the no. 1 tool for font work is pfaedit; I'd grab that and have a

I checked this last Friday. Both of them have a bullet symbol in them,
but I'm practically certain the font we're missing is a remapped
"Standard Symbols". Open up a copy of non-Ximianized OOo 1.1: go to
insert character, and check out the Standard Symbols font: you'll see
that it's exactly the same as URW Standard Symbols except that the
unicode mappings are different (e.g.: F0B7 instead of B7). In Ximian
OOo, there is _no_ such mapping (e.g.: we have B7 where we should have
F0B7)

Note, however, that the unicode points for the bullet in opens___.ttf
are identical between Ximian OOo and stock OpenOffice.org (there do seem
to be some minor differences elsewhere, which is worrying, but probably
isn't the source of this particular problem).
