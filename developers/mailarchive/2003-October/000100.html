> I have a quick question though the --with-distro tag, what is the used
> for?  If I say --with-distro=Lunar-Linux (my distro) the build fails
> with the mozilla error I posted a few days ago.  If I dont put anything
> it compiles fine. 

	If you use no --with-distro tag it uses the Ximian patch set [ this is
prolly what you want for maximum features ]. If you want to customize
your patch set you can use --with-distro=LunarLinux and edit
patches/OOO_1_1_0/apply to sub-set the patches for your particular
distro. If you're going to do that though; I suspect we're going to need
to expand how the patch sub-setting works currently to functionally
annotate the patches themselves and specify sub-sets differently since:

[ Debian, Ximian, Mdk, RedHat, LunarLinux ]

	is getting a bit unweildy; I suppose we should start doing functional
sub-setting ie.

[ GnomeVFS ]
[ Java ]

	or whatever, and have some map of requested tags.
