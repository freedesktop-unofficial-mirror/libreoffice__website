> It seems that in standard OOo, there's a font called "Standard Symbols",
> which is mapped to correspond to a private unicode codespace (F0xx). I
> _think_ what's going on is that standard OpenOffice.org is mapping the
> character C2B7 (given the font "Symbols") to this font w/ the unicode
> number F0B7. Hence perfect bullets every time. Is this correct?
>
> If so, my next question would be: where does this font come from? I
> can't find it anywhere in the OpenOffice.org distribution (I see the
> OpenSymbols font, but that seems to be used for something else). There's
> a Type1 font on my system in /usr/share/fonts/default/Type1 called "URW
> Standard Symbols", which seems to be exactly identical, except for the
> fact that it has different unicode mappings.

There isn't "Standard Symbols" in OOo. The only font coming with it
is "OpenSymbol" and is a TrueType font.

Note that another Symbol font is called "Symbol" (note "Symbol" and not "Symbols")
and is available from from Windows (symbol.ttf, in true type) or from Adobe Acrobat
Reader (Symbol or sy_____.pfb|pfa in Type1). And generally should be added
manually to OOo.

The fonts you are talking about is probably the font called
"Standard Symbols L" (this is the family name) and comes from
the URW fonts which are generally provided with ghostscript and
is also available to X11 and thus to OOo. Such font isn't
UNICODE encoded itself, and is a Type1 fonts. It has glyphs reencoded
in a vector with position from 0 to 255, so probably the Unicode re-encoding
is performed interally by OOo according to glyph names.

Speaking of such "Standard Symbols L" font, I also found a problem in using such font
when printing, see the bug issue #20595 on OOo IssueZilla.

Note also that using Symbol font is different than using "Standard Symbols L"
font, as unless one add a specific "Symbol -> Standard Symbols L" font replacement
(and I don't see it), the Symbol font is considered
"resident" when printing, as it's part of the Adobe Standard 35 PostScript font set,
while for any other font, it's downloaded/dumped in the PostScript file itself.
