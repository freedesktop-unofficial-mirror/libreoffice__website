Hi,

I continued porting C++ to C# (finished MapToManaged, ported
MapManagedType and MapUnoType (+ dependencies) which translate between
UNO typelib TypeDescriptions and managed System.Types). Also ported the
Cli_environment (a hashtable of UNO objects/their managed Proxies). 

Then I started with the UNO side proxy for managed objects. I'm trying
to keep most of the implementation of this in C# and have very thin glue
to provide the expected C interface.

I found out that my plan for climaker (assembly generator similar to
cppumaker, javamaker) had a bootstrapping flaw. I thought I could
rewrite it completely in C#, using the cli uno bridge. But the project
that builds climaker must be built before the bridge code. Will have to
find another plan when it's time to get climaker running.
