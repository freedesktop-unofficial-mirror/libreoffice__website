Hi dudie,

	So - I just split all the common options out in ooo-build into
Common.conf.in to save having so much rather opaque duplication of
configuration settings.

	Ark: you use --enable-libart instead of --without-gpc. I would
recommend using --without-gpc which uses an internal clipper instead of
GPC or an external libart. That is unless you know something I
don't ? :-) [ it'd be nice to share that option too ]

	Ark: you don't build with libstartup-notification ? why not ? it's a
tiny library with no problematic dependencies that implements
startup-notification [ ie. hour-glass cursor ] on startup, life will not
be so nice for users without this; recommended.

	Debian/Ark - we added the --enable-gnome-vfs option to configure OO.o's
built-in Gnome-VFS support (which is now up-stream), it's prolly a good
thing to turn on for eg. smb:/// share browsing.

	Finally, in patches/src680-m74/apply we have:

NonArkCommon: Evo2

	Which is rather ugly; I can understand the problems previously wrt.
different Evo. versions & all that - but now (thanks to Jayant) this
code will compile without any evolution libraries / development packages
present & will detect at runtime if it's available. So - any chance of
pruning that complexity back too ? :-)

	Thanks guys,

		Michael.
