Hello all,

I am a maintainer of the OOo related packages in SUSE. I cooperate closely 
with Michael Meeks actually. We are working on a package which combines 
features of the Ximian's and SuSE's packages. I hope that you don't get angry 
with my changes in the ooo-build. I am doing my best to do not break your 
work but...

I would like to inform you about the latest changes in the ooo-wrapper.in 
which is included in the ooo-build. I wanted to improve the compatibility 
with the former SuSE's wrapper. So I did:

* add the option --config-path to define another path for the user 
configuration
* add the options --enable-update and --disable-update which allow to change 
default behaviour within an update; the update is still enabled by default; 
it makes sense to disable it if the user shares the configuration over net 
(NFS home) and uses more versions of OOo in parallel
* add new option --icons-set to select another set of icons
* do not longer insist on one exact path to the user configuration
* use the default path like $HOME/.ooo-1.1 instead of $HOME/.xopenoffice1.1;
the RedHat's default path remains unchanged
* use an alternative default path like $HOME/.ooo-1.1.2 if the update is
disabled and the default path is already in use
* do not longer use /etc/openoffice/autoresponse.conf but generate the 
autoresponse file in /tmp by occasion
* process two more links (ooweb, oofromtemplate)

I am open to find a common solution if you strictly do not agree with a 
particular change.
