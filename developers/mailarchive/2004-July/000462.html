On Thursday 29 July 2004 17:01, Arkadiusz Miskiewicz wrote:
> On Thursday 29 of July 2004 16:44, Petr Mladek wrote:
> > Hello all,
> >
> > I would like to improve the generated file lists and mark the files
> > which can be configured by users.
>
> First of all configuration files should not live in /usr or any other
> directory beside /etc (at least that's what we do at PLD, keep
> configuration in /etc).

Fully agree.

> Maybe symlinks pointing from /usr/lib/..../somefile to
> -> /etc/openoffice/conf/somefile and mark
> %config /etc/openoffice/conf/somefile?

Yes, it would be one hackish solution.

The problem would be with directories. I think it would be fine to link the 
entire directories like share/config or share/registry from /etc but there is 
problem with update. RPM does not support to replace a directory with a 
symlink within an update.

> Also such configs should be per-user not system wide but probably that
> can't be easily fixed so this is better that than nothing.

I think that this is not the problem. The system configuration defines the 
default setting. The user can change it in his personal configuration. I 
think that this approach is the same for any other software.

IMHO the problem is that the configuration in not on one place.

> But that's not so important for us right now so we will follow the way you
> choose.
>
> > I am afraid that we do not find a compromise. So, I would suggest to set
> > a vendor dependent variable in bin/setup. Something like that:
> >
> > if test "z$DISTRO" = "zNLD" -o "z$DISTRO" = "zSUSE"; then
> > 	RPM_CONFIG_FLAG="%config"
> > elif test "z$DISTRO" = "zPLD" ; then
> > 	RPM_CONFIG_FLAG="%config(noreplace) %verify(not md5 size mtime)"
> > fi
>
> I'm ok with this.

OK, I'll do what I proposed ;)
