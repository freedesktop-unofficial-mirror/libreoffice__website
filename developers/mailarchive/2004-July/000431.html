I think the list of issues in the apply file is a very good starting
point to go from as the list of outstanding patches.  The non-numbered
patches fall into a few groups:

 - Not suitable for all arches / OSes

 - Only make sense on a particular distro/subset of distros 
   (Those in XXXXOnly sections)

 - Experimental in nature, with known bugs or not fully tested.  It
   might help to mark them as such, or create an issue anyway.

 - Highly experimental patches :) Generally, those in unused sections
   towards the bottom of the apply file.

 - Patches which haven't had issues filed for them yet, but could be
   included upstream.   Actually, although Michael doesn't like the
   issue filing process, he's probably created issues for more of his
   patches than I have *blush*.  I still have to create issues for
   several of my patches.
