> * Do we have OOo issues for the patches ? Michael, I know you find it 
> someway bureaucratic, but it makes my life of writing release notes more 
> easy and exact (also QA will be happy if they are able to track the 
> milestones).

	I try to file issues per patch; especially for the ones that can
usefully go up-stream easily. However, the practice varies amongst
people working on ooo-build, and we're not too anal ;-) best practice as
codified here: http://ooo.ximian.com/ooo-build.html#section-2.4 is to
commit to IZ before CVS.

> If the answer is no, how can we set up a plan, what needs to get in OOo 
> and what not. Is everybody aware of the current plan for 2.0 (UI freeze 
> end of July, Beta in October) or does anybody need a change of the plan ?

	Well - we hope to be working on the 2.0 branch more actively in a few
weeks/months. Whether the bugs get up-stream or not is _really_ more a
function of whether the Sun / IZ process is at all workable - so far,
without periodic kickings of backsides by interested parties [ Stefan /
you ] almost nothing would be merged up.

	OTOH, some of the bug numbers in the patches list, are things that have
been applied to HEAD, and will not be applied to stable; others are
plain back-ports of HEAD features; others have been (already)
gratuitously re-written up-stream, and we're waiting for someone to do
the work comparing / selecting / re-back-porting the new work. Some
other patches belong to essentially vendor-specific stuff, eg. strike-
versioning, branding patches etc. and yet others belong to in-progress
feature development.

	Anyhow - in total, it's great to have your interest / attention. The
best place for the latest patch / issues list is:
	http://ooo.ximian.com/ooo-build/patches/OOO_1_1_2/apply

	Hope that helps,

		Michael.
