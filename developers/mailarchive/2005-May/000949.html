Hi,

I've seen a change of policy regarding the naming of the patches that need 
fixing in the new milestones. I had the feeling that the newest patches are 
without the milestone suffix, the older ones get -mXX.diff when the patch has 
to be updated for the new milestone, so that we still can track the changes 
in the CVS easily. But now I see e.g.:

[ TemporaryHacks < src680-m102 ]
# temporary hack to avoid the warning about missing return values in gcc4
warning-return-values-stlport.diff

[ TemporaryHacks >= src680-m102 ]
# temporary hack to avoid the warning about missing return values in gcc4
warning-return-values-stlport-for-m102.diff

Here warning-return-values-stlport-for-m102.diff is the newest patch, tracking 
of the changes will be more difficult when the < src680-m102 vanishes.

So, do we really want this?
