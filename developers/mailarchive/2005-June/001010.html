I can now confirm that --with-java indeed fixes the "file picker will open 
over and over again on Save As" bug (btw, if anyone else is wondering, in 
order to build --with-java without installing proprietary code, you have to 
use gcj 4.0.x with a patch to backport some class library fixes from gcc HEAD 
branch in CVS. You can also patch gcj 4.0.x to work with gcc 3.4.x if you 
can't use gcc 4.x for other reasons.
