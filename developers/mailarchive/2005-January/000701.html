I'm currently trying to build 1.9.70 on a Linux x86 machine with gcc 3.4
and want to report some problems I've noticed. My build configuration is
pretty much like the default one besides having disabled java and kde
widgets.

* pyuno/source/loader: the build breaks with undefined references to
python::Runtime::Runtime among others. Should perhaps in makefile.mk
the .ENDIF line below PYUNOLIB=... be moved above the .IF "$(GUI)" line
so that the PYUNOLIB variable gets defined when using system-python,
too? (not yet tested)

* pyuno/source/loader: after circumventing the above problem, the build
breaks when trying to register reflection.uno with the error
terminate called after throwing an instance of
'com::sun::star::registry::InvalidValueException'
Don't know whether this problem has the same origin as the one above,
doesn't look like.

* connectivity/source/drivers/evoab1.5: the build breaks due to
undefined references to functions of gobject-2.0. 
Please merge the changes in
ooo-build/patches/evo2/connectivity-source-drivers-evoab1-5-makefile-mk
from ooo-build-1-3 (1.2 -> 1.2.6.1) to HEAD
http://cvs.gnome.org/viewcvs/ooo-build/patches/evo2/connectivity-source-dri=
vers-evoab1-5-makefile-mk.diff?r1=1.2&r2=1.2.6.1
