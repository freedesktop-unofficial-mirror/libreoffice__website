Carlos,

Ximian's OOo does not use psprint, and therefore the settings in
psprint.conf have no effect.  Historically, psprint has provided two
things:  printer configuration, and font addition.

Printer Config:  its now done completely by CUPS. You do not need psprint
at all.  If you have added a printer via your OS (in Fedora its in the
menus, used to be redhat-config-printer, now its system-config-printer),
it will be available to OOo.  Therefore, psprint is not needed for
printers.

Font Addition:  its now done completely by Fontconfig.  If you add fonts
to the standard directories, like /usr/share/fonts, they will be
available to OOo.  Therefore, there is no need for psprint/spadmin for
managaging fonts.

Print to PDF:  now, you should use File->Export to PDF.  PDF export is
built into OOo and you no longer need to do anything in psprint.conf to
get it.

So, you see the 3 main uses for psprint/spadmin are available by more
integrated means.  It really makes no sense to have 2 different ways to
configure printers and fonts each, one in the system and one in OOo.

Dan
