> Where should I look to understand the mechanism for text layout?

	I believe you want to see 'sallayout' and it's implentations in
vcl/[unx/win]/source/gdi/sallayout.cxx or somesuch.

> Tamil unicode fonts are experiencing a trauma due to non standard
> encoding history. My immediate task is to filter the good ones out.

	Ah - if it's a font problem - the solution is perhaps easier; the font
code is _highly_ complex; but it's possible you can fix the problem with
a simple schema fix. The best way to prototype this is to edit a live
install's file mine is:

	/usr/lib/ooo-1.1.0/share/registry/data/org/openoffice/VCL.xcu 

	And update the mappings to suit your paticular locale best. If you can
do that, send in a patch and we can include it in ooo-build / help get
it up-stream.
