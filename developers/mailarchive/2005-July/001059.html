On Sat, 2005-07-16 at 12:54 +0200, Bernhard Rosenkraenzer wrote:
> Hi,
> I got all recent milestones to compile with gcj and --with-java, but 
> post-install, it refuses to find gcj as a JRE (Options -> Java shows a 
> checked "Use a Java runtime environment", but no available JREs, and it won't 
> even offer gcj if I try to add it manually).
> 
> Any ideas, or pointers to which file in the code to check? (Did I mention 
> OOo's code is way too large? ;) )

Start looking in jvmfwk, e.g. jvmfwk/plugins/sunmajor/pluginlib.
There the various jvms are searched for, see if it goes looking for gcj,
the gnujre.cxx. "PostInstall", implies that it was found during actual
installation, there is a different code patch in stoc I believe which
calls the jvmfwk stuff at registration time vs normal run-time. So
assumng that the java components were registered it's got to be
something relatively trivial
