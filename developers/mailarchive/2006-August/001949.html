Hi,

I went through the list of the issues we cover in ooo-build, and I found out 
that there's a number of patches for which we do have an issue in IssueZilla, 
but it is already closed - this way it'll never get out of our patchset ;-)

I tried to fix the situation, but there's too much patches there to fix it 
myself; so I created a script 'show-issues' (under bin/) that lists the 
issues according to the maintainers.  (It excludes patches from 
FixesNotForUpstream and Defaults section.)

Please - have a look at the list of your issues.  For the duplicates, update 
the issue number in the apply file so that it points to the original issue, 
not the duplicate.  For the 'closed fixed' issues, check that we still need 
the patch - if yes, create a new issue for it, if no, just bin the patch...  
For the new/started issues that weren't updated for a longer time, ping the 
upstream maintainer ;-)  Thank you!
