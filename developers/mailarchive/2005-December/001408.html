So, the documentation <http://udk.openoffice.org/cpp/man/cpp_bridges.html>
says that it's an inout parameter. And the implementation of
registerProxyInterface (cppu/source/uno/lbenv.cxx) treats it like that. It
never stores the ppInterface parameter (which is &proxy), but always uses
*ppInterface, and possibly assigns a different proxy to *ppInterface (after
freeing/unrefing the old value). Still nervous?
