Hi,

I changed the old icon switchitching patches for a snapshot of 
'iconswitching1' CWS for the milestones m141 and newer.  The functionality 
should be the same - but there is one incompatibility.

'iconswitching1' uses name of the theme as the value in the configuration file 
when storing the settings.  The old patches were using its index, which can 
result in an error during startup (corrupted user configuration).  This 
should not affect users, because we were using '.ooo-2.0-pre' as the user's 
directory for 2.0, and now we have switched to '.ooo-2.0'.  The developers 
that install OOo from HEAD regularly should just remove their config 
directory.

If you used another user's directory in your distro (like '.ooo-2.0' already 
for the 2.0), you'll probably have to change it to something like 
'.ooo-2.0.1'.  Sorry for the inconvenience :-(
