Hi list!
I did a patch to disable xalan completely.

This patch is useful for those where caching of xalan.jar OOM-kills their system.
At the moment they have to disable the gcj-dbtool feature to get around this.
With this patch they should be able to use the gcj-dbtool feature.
Without xalan we need xsltproc to do its job.

After applying this patch you have to set:
--enable-xsltproc --disable-xalan

Module rhino built fine without xalan.
No idea if module beanshell needs it.
I use it from system.

I do not know if module filter is ok.
To use xsltproc for PACKLANG and
to use java for MERGE.

I am still building OOo2. The patch may be incomplete.
I missed some important things, for sure.
I hope to get some feedback or comments! TIA.
