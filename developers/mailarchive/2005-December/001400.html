Hi all,

I tracked down my menu font problems. Looks like Tahoma.ttf is badly displayed 
by OpenOffice 2.0.1. Lets see with 3 screenshots:

1- http://cekirdek.uludag.org.tr/~ismail/ooo-bad.png , run as 
SAL_USE_VCLPLUGIN=kde , OpenOffice.org m143 snapshot

2- http://cekirdek.uludag.org.tr/~ismail/ooo-bad-vcl-gen.png , run with 
SAL_USE_VCLPLUGIN=gen ./swriter , OpenOffice.org m143 snapshot

3- http://cekirdek.uludag.org.tr/~ismail/ooo-good.png , run as 
SAL_USE_VCLPLUGIN=kde , OpenOffice.org 2.0.1 release 

All snapshots were taken with Tahoma font in use. strace shows OpenOffice.org 
successfully opens Tahoma.ttf and fc-match agrees :

cartman at pardus ~ $ fc-match -v Tahoma|grep file
        file: "/usr/share/fonts/windows/tahoma.ttf"(s)

same file in strace is opened. So is there any patch to suspect? FWIW here is 
the configure line thats used :

./configure --prefix=/opt/OpenOffice.org --sysconfdir=/etc --with-lang=tr --disable-post-install-scripts --with-installed-ooo-dirname=ooo-2.0 --enable-h 
unspell --disable-gtk --with-jdk-home=/opt/blackdown-jdk-1.4.2.02/ --disable-cairo --disable-mono --with-gcc-speedups=ccache --with-distro=Pardus --disable- 
access --with-tag=src680-m143                                                                                                                               

Pardus.conf.in attached which is hooked up in configure.in.in . I appreciate 
any ideas, but a temporary solution is to set a rule to replace Tahoma font 
with some other font in font config dialog.
