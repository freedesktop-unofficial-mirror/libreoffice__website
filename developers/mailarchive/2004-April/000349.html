Hi Dan,

	Just watching the windows build - I see a lot of 'makedepend'
statements taking a lot of time for each directory ( they are invisible
under Linux ), and I got to wondering how much time/space they use; so
for vcl here it is:

	  size with    deps  28407719
	  size with no deps  26733912
	* saving              1673807 = ~1.5Mb

	  time to re-build deps:    12
	  time to re-build nothing: 2.5
	* saving                   ~10 seconds

	So - I guess there is prolly a good way to count the size of all
the .dpcc files in a tree.

	Dependencies are really unnecessary for a straight-through build; and
if you simply export nodep=1 - you don't get any; we have to delete them
on 'bin/relocate' anyway (for hacking) since they have hard-coded paths
in them :-) so all in all, this is a nice win.

	I've added export nodep=1 to ooo-build; I guess the Debian scripts
might want some more tweakage. Should save some time / space.

	HTH,
