Hi Anil,

	So - in addition to the various minor tasks / bug fixing that goes on
I'd like you to be working towards making this happen. The big picture
is that we require the gtk+ HEAD file selector (gtkfilechooser) to be
used instead of the native OO.o file selector. Federico is working on
the gtkfilechooser and assures me that the API is equivalent and will
cause no problems.

	Splitting this down this means you should:

	+ build HEAD glib, pango, atk, gtk+ and install them
	+ get familiar with their API / how they work
	+ investigate and document the OO.o file selector code
	  in openoffice/doc/filepicker.txt - there is already a chunk
	  of research / pointers in there.
	+ implement a file selector component for gtk+ in the same
	  style that the win32 component does this
	  (in fpicker/source/win32/filepicker)
	+ investigate completing the gtk+ mainloop integration cf.
	  the TestGtkIntegration section in 'OOO_1_1_0/apply'
	+ investigate cross-toolkit VCL <-> Gtk+ modality issues -
	  the file selector is modal.

	This is a longish term task :-) I'm hoping we can make good progress on
this in the next few months though.

	Thanks,

		Michael.
