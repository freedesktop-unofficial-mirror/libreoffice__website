Hello,

there is error in libevo2.so SDBC driver. Driver effectively disables =
other drivers to make a connection. This is caused by OEvoabDriver::accepts=
URL() method which always returns sal_True even for URLs which are / have =
to be handled by other drivers.

Resolution of this bug is to modify acceptsURL() method to return sal_True =
only for URLs which belong to Evolution driver (as it's required by SDBC =
specification).

Some SDBC drivers aren't affected by this bug (for example dbase driver) =
however these drivers are probably asked for URL acceptance by DriverManage=
r before libevoab2 driver. PostgreSQL driver (http://dba.openoffice.org/dri=
vers/postgresql/index.html) is affected by this error.

I have seen this behaviour on SuSE 9.2.
