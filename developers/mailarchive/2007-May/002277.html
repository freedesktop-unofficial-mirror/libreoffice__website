Hello, good people,

I am now working sporadically whenever I have some dead moments on an
integration of the import filter for WordPerfect graphics into ooo-build.

It will add another tarball (libwpg) into ./download. I will code also
the possibility to use the libwpg as system library (for coherence and
completeness reasons), nevertheless, I would beg the distribution
packagers not to distribute CVS snapshot of libwpg in their
distributions. The library is API unstable and would really not like to
have incompatible binaries hanging around. I am likely to rewrite
completely the PainterInterface class as soon as I have a good idea
about what could be the best for all the uses libwpg is destined to
(standalone wpg converter and possible plug-in for libwpd to convert
images inside wpd documents).

If I have time, the work should be finished at the end of this week.

Cheers and thanks for your understanding

Fridrich

P.S.: Rene, if you decide to disregard my kind request :-), rename the
library to something like 0.0.1 so that it produces *.so.0.0.0. The
first version I will officially release will be libwpg-0.1.0. But be
warned that I will not take any care of API or ABI compatibility between
unreleased CVS snapshots.
