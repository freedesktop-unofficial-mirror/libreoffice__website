>to the user settings, etc.

Yes. That's exactly how libdadapt is supposed to work. For instance the 
"file" module is for
transparent network file I/O and filechoosers:

libdadapt_file_kde_1.so
libdadapt_file_gnome_1.so

(That's the one i'm trying to write a prototype for)

libdadapt.so itself just provides three functions (it is just a dumb 
wrapper to dlopen and dlsym, 50 lines of code).

DaModule * daModuleOpen(const char * name);  // name = module name, 
"file" for instance
DaInterface * daModuleGetInterface(DaModule *, const char * interfaceName);
int daModuleClose(DaModule *);

The modules just provide the single entry point 
"daModuleGetInterface_impl" which returns a struct of function pointers 
(that are defined in a defined in "dadapt_mod_xyz.h".)
