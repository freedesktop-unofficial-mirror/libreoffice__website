On Tue, 2005-11-29 at 09:21 -0800, Leibowitz, Michael wrote:
> I forgot to explain that for ooo-build, I want the option to have a
> different default.  This option is for which malloc is being used.  For
> ooo-build, I want to make the default allocator ptmalloc2 (glibc).  It
> is marginally faster on recent machines, and makes debugging easier.
> However, the oo.o configure will keep oomalloc as its default.

	Ah - that's pretty easy to do.

	Just add the same configure option to both external & internal
configures; if the option is not set in the external one - propagate the
default you wish for ooo-build (cf. OOO_WIDGET_FLAGS & bin/setup.in), if
not set in the internal configure default to the option for up-stream
OO.o
