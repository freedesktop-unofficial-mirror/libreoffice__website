Caolan McNamara schrieb:
> --without-stlport4 will use system stl if you have gcc, but it will
> break with visibility at runtime, see gcc#19664#

The Debian package doesn't use --without-stlport4/gcc's stl.

It uses --with-stlport4=<path to a tree that links to the headers from 
debian's stlport package>:

# outside the ooo tree, but in debian's build tree
mkdir -p stlport4;
ln -sf /usr/include/stlport stlport4/stlport;
ln -sf /usr/lib stlport4/lib

and

configure ... --with-stlport4=$(CURDIR)/stlport4 ...
