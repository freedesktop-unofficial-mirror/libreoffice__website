On Friday 01 April 2005 15:28, michael meeks wrote:

> 	The new apply file lives in patches/src680/apply, the syntax extension
> should be obvious from reading it - it's like this:
>
> [ NovellOnly, RedHatOnly < src680-m79 ]   or:
> [ NovellOnly, RedHatOnly > src680-m79 <= src680-m89 ]  etc.

An idea came to my mind over the weekend: Wouldn't it be a bit more convenient 
to have the milestones connected with the patches instead of the sections? 
Like:

[ NovellOnly ]
some-patch-m79.diff <= src680-m79
some-patch.diff >= src680-m80
other-patch-that-fits-all.diff

I am afraid that we could end up with a section for every single patch. But 
maybe this milestone-after-patch approach has drawbacks as well :-)
