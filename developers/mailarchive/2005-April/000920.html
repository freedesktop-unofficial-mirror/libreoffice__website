IIRC, the new behaviour is to omit dependent libraries if pkg-config
thinks the system supports inter-library dependencies.

That is, if gconf depends on ORBit2 and glib then linking against
libgconf will pull in ORBit2 and glib by reference.

If it is making the wrong decision about the platform's support for
indirect dependencies, I'd suggest filing a bug at
https://bugs.freedesktop.org.  Tollef has been pretty good about fixing
the bugs I've found.

Make sure you try the latest version first though (0.17.2).

>	Before I try to get some odd-looking change up-stream here for all our
>pkg-config usage, it'd be nice to have some rational/authoritative
>explanation for why pkg-config has "gone autotools" in it's backwards
>compatibility strategy.
>  
>
Some of the breakage can be attributed to the long period of time
between releases of pkg-config.  Some changes since 0.15 had been
sitting in CVS for two years without wide testing.

If you find something obviously broken in the new releases, report it as
a bug.  I don't think the policy is to treat all incompatibilities as
"applications using undocumented APIs" (0.17.2 fixes a number of
problems that occurred with the 0.16 release).
