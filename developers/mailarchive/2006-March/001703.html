On Sun, 2006-03-12 at 01:41 +0100, Martin Kretzschmar wrote:
> Hi,
> 
> This bug
> 
>   https://launchpad.net/distros/ubuntu/+source/openoffice.org/+bug/33786
> 
> seems to happen because of a missing glyph (0x2212, between summation and
> minus/plus sign). This is missing in our opens___.ttf, but not in
> OpenSymbol.sfd.
> 
> How can I get the minus back into opens___.ttf or how do I generate the ttf from
> the sfd?

You can use fonforge (be sure to use cvs version) to generate ttf font
from sfd source.

I looked at the 0x2212 glyph and there was a problem with unicode value
set to 0x00ad, so the generated ttf file was missing 0x2212 minussign
glyph. Looks like someone added minussign to the font, but didn't set
unicode value correctly.

It is now fixed in cvs.
