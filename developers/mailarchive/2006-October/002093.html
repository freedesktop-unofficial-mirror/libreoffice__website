well, it's actually /usr/lib/jvm/java-1.4.2-gcj for me, it's a symlink.
what the symlink points to is almost certainly distro dependant. 

If "/usr/lib/jvm/java-1.4.2-gcj" was common across distros (I don't
know, it might be as I assume it came from jpackage naming schemes) then
we might be able to default to this jdk-home if one is not set and we
have detected from configure that we are running a gij which is a high
enough version to come with libjvm.so. If it was fairly likely to exist
on fedora (yes), debian and other java-gcj using distros then that would
make building OOo with gcj more pain-free.
