I have already heard about this problem. File systems do not support file 
names longer than xxx bytes (I think that xxx is 256 on many file systems). 
The problem is that OOo produces one very long file name during installation 
when --with-lang=ALL is used.

I did a fast look into the sources but I did not find a simple fix. I think 
that there is no reason to use such a long file name for a log file and it 
should be fixed.

As a workaround you can either remove some localizations from the list in 
OOO_2_0_0/solenv/inc/postset.mk (it was enought to remove 2-3 localizations 
three weeks ago) or you could use --with-lang="list of really required 
localizations"

Note that many localizations are not finished and are not useful now anyway. 
We currently use:
--with-lang="en-US af ar be-BY bg ca cs cy da de el en-GB es et fi fr gu-IN hu 
hr it ja ko nb nl nn pa-IN pl pt pt-BR ru sk sl sv tr vi xh zh-TW zh-CN zu"
