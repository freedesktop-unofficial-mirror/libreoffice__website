Not a major change in the main code: just added a facility to compile
a separate UNO package from the same source tree.  All changes went
into scsolver/workben.

Though one change I did yesterday may be of importance: it prevents
nested two-phase search, which was done in reponse to one user
reporting a case of infinite loop during initial search, thus freezing
the whole OO.o.  After the change I put in yesterday, the solver
properly fails with a dialog when it detects a nested two-phase
initial search, instead of going into an infinite nested two-phase
(this should never happen in theory).
