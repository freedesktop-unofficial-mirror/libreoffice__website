Hi,

I started to play with qatesttool. It found some crashes with gcj/gij. I 
wonder if you see this crashes as well of if I do something wrong.

A. Package Manager. How to reproduce the problem:

1. start ooffice
2. Select Tools/Package Manager...
3. Press the button Add
4. Select the attached capitalise.uno.pkg
5. Press the button Open

Then I get segfault. It is 100% reproducible with fresh OOo configuration and
when gcj/gij is used as the JRE.

It does not happen with another package Basic_Addon.zip. Though, after you try
to add the problematic capitalise.uno.pkg, the OOo UNO-package database is
somewhat broken and OOo crashes when you try to access Tools/Package
Manager.../My Packages.



B. Crash with XHML export. How to reproduce:                                                               
                                                                                
1. Start oocalc                                                                 
2. Check Tools/Options/Java to make sure that you use JRE from the vendor       
   "Free Software Foundation". It is the gcj/gij                                
3. Enter some random text                                                       
4. Select File/Export...                                                        
5. Select File format XHTML (.xhtml)                                            
6. Enter a file name                                                            
7. Press the button Export                                                      
                                                                                
I got a crash.                                                                  
                                                                                
It works with Sun JRE.                                                          



C. Macros. How to reproduce:

1. Use gij/gcj                                                                  
2. Open qatesttool/framework/first/input/10erTest_680.sxw                       
3. Enable macros                                                                
4. Select Tools/Macros/Run Macro...                                             
                                                                                
I got a crash.
